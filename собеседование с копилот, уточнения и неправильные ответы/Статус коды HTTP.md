Что означают статус‑коды HTTP?:

- **200**
    
- **400**
    
- **404**
    
- **500**

### ✔ 200 — **успешно**

Да, это означает, что запрос выполнен без ошибок.

### ❗ 400 — **не просто “не выполнен”**

400 — это **ошибка клиента**. Обычно означает, что запрос был неправильным:

- неверный формат данных
    
- отсутствуют обязательные поля
    
- некорректный JSON
    

### ✔ 404 — **не найдено**

Да, это ошибка: сервер не нашёл ресурс. Например, страница или API‑эндпоинт не существует.

### ❗ 500 — важно знать

500 — это **ошибка сервера**. То есть проблема не в клиенте, а в самом сервере:

- баг в коде
    
- упала база данных
    
- сервер не смог обработать запрос
    

# ✔ Как выглядел бы идеальный ответ на собеседовании

- **200 OK** — запрос успешно выполнен
    
- **400 Bad Request** — ошибка клиента, неправильный запрос
    
- **404 Not Found** — ресурс не найден
    
- **500 Internal Server Error** — ошибка на стороне сервера
