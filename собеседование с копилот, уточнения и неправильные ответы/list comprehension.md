Что такое **list comprehension** и чем он отличается от обычного цикла `for`?

### ❗ Что неверно / требует уточнения

- **Обычный цикл тоже создаёт новый список — ровно один раз.** Разницы в количестве создаваемых объектов нет.
    
- **List comprehension не создаёт “новый объект при каждом вызове программы”** — он создаёт список ровно один раз, когда выполняется, точно так же, как цикл.
    
- Разница не в количестве объектов, а в:
    
    - **скорости** (list comprehension быстрее)
        
    - **читаемости**
        
    - **компактности**
        
    - **возможности встроить условия прямо внутрь выражения**

**List comprehension** — это компактный синтаксис для создания списка на основе итерируемого объекта.

Пример:
```python
squares = [x*x for x in range(10)]
```
### Отличия от обычного цикла:

- короче и читается легче
    
- работает быстрее, потому что реализован на уровне C
    
- позволяет встроить условия прямо в выражение
    
- создаёт список в одной строке, без явного `.append()`
    

Обычный цикл:
```python
squares = []
for x in range(10):
    squares.append(x*x)
```
